# 非機能セキュリティ方針（P0-6）

最終更新: 2026-04-25

## 目的

MVP段階で最低限守るべき、PII・通信・秘密情報の保護方針を定める。

## 対象範囲

- `Staging / Production` の2環境
- 認証（メールOTP）とカップル連携データ
- API・バッチ・運用手順

## 1. PII（個人情報）保護

- 最小収集: MVPでは `email` と `display_name` 以外の個人情報は原則収集しない。
- 目的限定: 収集項目は認証・ペア連携・利用分析のみに使用する。
- 表示制御: 週次公開前のニンジャログは相手に見せない（仕様準拠）。
- ログ方針: メールアドレスやOTPを平文ログに出力しない。
- 保持方針: 退会/解除時のデータ削除・匿名化方針は `P2-2` で規約と整合させる。

## 2. HTTPS/TLS方針

- 本番は HTTPS 強制（HTTPからHTTPSへリダイレクト）。
- `Secure`/`HttpOnly`/`SameSite` 属性付きCookieを必須にする。
- 外部API連携もTLS前提とし、平文通信を禁止する。
- Mixed Content を許容しない。

## 3. 秘密情報（Secrets）管理

- Secretsの正本はCloudflare環境側に置く（ADR 0001方針に準拠）。
- リポジトリに `.env` や資格情報をコミットしない。
- ローカル開発用秘密情報は個人環境に閉じ、共有はしない。
- CIで必要なSecretsは最小権限・最小個数で管理する。

## 4. 認証まわり最低ガード（MVP）

- OTP有効期限 10分以内（ADR 0002）
- OTP再送クールダウンあり
- OTP試行回数制限あり
- 認証イベント監査ログを保存

## 5. 運用ルール

- 依存パッケージは定期更新し、脆弱性を確認する。
- 権限変更・Secrets更新は履歴が残る手順で実施する。
- インシデント時は「検知→一次封じ込め→復旧→再発防止」を記録する。
