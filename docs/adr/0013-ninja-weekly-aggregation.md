# 0013. ニンジャ週次集計（Asia/Tokyo 週・冪等ジョブ）

## ステータス

承認済み

## コンテキスト

- 開発計画書 P1-5 で、ニンジャの **週次集計**を「手動トリガーまたはバッチのたたき」で実装する。
- 企画書では週末の発表イメージがあるが、MVP では **クラウドのスケジューラ本番配線**より **集計ロジックと冪等性**を優先する。

## 決定

### 週の区切り

- 集計の基準タイムゾーンは **Asia/Tokyo** とする。
- 週は **月曜 00:00 JST から次の月曜 00:00 JST 未満**（半開区間）とする。`week_start` はその週の **月曜のカレンダー日**を `YYYY-MM-DD` で保持する。

### 集計エンティティ

- `ninja_weekly_summaries` に、カップル単位・週単位で **オーナー／パートナー**（`couples` の `memberIds` の先頭をオーナー、2 人目をパートナーとみなす）の **合計ポイント**と `published_at` を保存する。
- `UNIQUE(couple_id, week_start)` で 1 週 1 行。同じ週へジョブを再実行しても **上書き再計算**し、冪等とする（申告が後から増えた場合も再 publish で追随）。

### 公開の意味

- `published_at` が埋まった行が存在する週について、API は **双方の合計ポイント**を返す（個別申告行は引き続き非公開。0012）。
- 未公開週では **自分の合計のみ**返し、相手の合計は `null`（非公開）とする。

### ジョブの起動

- `POST /ninja/jobs/publish-week` で集計を実行する。リクエストボディに `weekStart`（月曜の `YYYY-MM-DD`）を指定できる。省略時は **現在时刻を含む週**の `week_start` を対象とする。
- **不正な呼び出し防止**: 本番では環境変数 `NINJA_PUBLISH_SECRET` を設定し、`Authorization: Bearer <secret>` を要求する。未設定かつ非本番では、ローカル開発・スモークのために認証を省略できる。
- Cloudflare Cron などへの接続は後続（Worker の `scheduled` ハンドラやダッシュボード Cron）は本 ADR の範囲外とし、**エンドポイントさえあれば手動・外部 Cron から叩ける**状態を MVP の完了とする。

## 結果

### メリット

- 週境界が明示され、集計と再実行の説明がしやすい。
- 冪等ジョブにより運用ミス・再デプロイ時の再実行に強い。

### トレードオフ

- 企画書の「日曜 20 時」など細かい公開時刻は MVP ではジョブ実行タイミングに委ねる（時刻そのものは `published_at` に記録される）。
