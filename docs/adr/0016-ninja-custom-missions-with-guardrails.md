# 0016. サイレント・ニンジャのカスタム任務（制約付き自己設定）

## ステータス

承認済み

## コンテキスト

- 企画書 §4.2 では、サイレント・ニンジャに「カスタム任務登録」を含める構想がある。
- 一方で [0012](./0012-silent-ninja-declarations.md) は MVP の公平性と実装簡素化を優先し、任務を API バンドルの静的定数のみ有効としていた。
- 実運用を想定すると、生活スタイルに合わせた任務をカップル自身で追加できる余地がないと、継続率と納得感が下がる懸念がある。
- ただし自由入力を無制限に許可すると、点数の恣意性や重複任務の乱立で比較の公平性が崩れる。

## 決定

### 0012 の更新範囲

- [0012](./0012-silent-ninja-declarations.md) の「任務カタログはサーバー静的定数のみ有効」を部分置換し、**静的任務 + カップル内カスタム任務**を有効とする。
- 0012 のうち、申告ログ構造・相手非通知・認可方針は引き続き維持する。

### カスタム任務の制約

- カスタム任務は **作成者の active couple 内のみ有効**とし、他カップルには共有しない。
- カスタム任務のポイントは自由入力にせず、**許可済み固定値（例: 5pt / 10pt）**から選択のみ可能にする。
- カップル単位で **週あたり作成上限**（例: 3 件）を設ける。
- 任務作成後のポイント変更は不可とする（名称の軽微修正は別途仕様化）。

### 申告時の整合性

- `ninja_logs` には既存方針どおり `mission_id` と `point` を保存し、申告時点の点数スナップショットを監査用に保持する。
- サーバーは `mission_id` が「静的任務または当該カップルのカスタム任務」であることを検証し、クライアント送信値で点数を確定しない。

## 結果

### メリット

- 生活実態に合わせた任務設計ができ、継続利用と当事者感を高められる。
- 点数固定・作成上限により、自由度を持たせつつ公平性の下限を守れる。
- 0012 の監査可能なログ方針を維持できる。

### トレードオフ

- 任務の永続化先（新規テーブル等）とバリデーション実装が増え、MVP の単純性は下がる。
- カスタム任務の文言重複や品質管理は、将来的に正規化・提案機能の検討余地が残る。
