# 0002. MVP認証方式をメールOTPにする

## ステータス

承認済み

## コンテキスト

- `P0-4` で認証方式の意思決定が必要。
- デプロイ基盤は Cloudflare（[0001](./0001-deployment-platform-cloudflare.md)）で確定済み。
- MVPでは実装速度と運用シンプルさを優先し、カップル連携フロー（招待コード）までを確実に動かしたい。

## 決定

- MVPの一次認証は **メールOTP（ワンタイムコード）** を採用する。
- ソーシャルログイン（Google/Apple等）は **1st強化** で追加検討とする。
- セッションはサーバー側で管理し、`Staging / Production` の環境ごとに分離する。

## 結果

### メリット

- UI/導線がシンプルで、MVPの実装と検証が速い。
- Cloudflare環境でも構成が分かりやすく、運用手順を統一しやすい。
- 端末依存が少なく、β検証対象に配布しやすい。

### トレードオフ

- メール到達性に依存するため、送信基盤の品質確保が必要。
- ソーシャルログインに比べ初回入力の手間がある。
- OTPの有効期限・再送回数・レート制限を適切に設計しないと不正利用リスクがある。

## 実装時の最低ルール（MVP）

- OTP有効期限: 10分以内
- 再送制限: 連続再送にクールダウンを入れる
- 試行回数制限: 一定回数で一時ブロック
- 監査ログ: 認証成功/失敗イベントを記録
